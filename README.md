# C-Adjusted GDP Explorer

**What is the world actually earning?**

A country's GDP counts every dollar produced — including dollars generated by industries whose external costs exceed their private revenues. C-Adjusted GDP subtracts those system welfare losses, producing a measure of *net* economic output: what remains after accounting for the damage that conventional GDP ignores.

This tool applies the System Asset Pricing Model (SAPM) across 190 countries and 30 domains. Every parameter is documented. Every source is cited. Fork the repo. Change a β_W. The instrument is the argument.

## Theoretical Foundation

### The Private Pareto Theorem (Postnieks, 2026a)

The bilateral payoff space — the space of outcomes Pareto-optimal for the transacting parties — structurally excludes system welfare. A transaction can be simultaneously Pareto-optimal for A and B and welfare-destroying for the system C. No bilateral contracting mechanism can resolve this; it is a structural impossibility, not a market failure amenable to Pigouvian correction.

**Formal statement:** Let Ω = {(a,b,c) ∈ ℝ³ : a≥0, b≥0} be the full welfare space. The bilateral Pareto frontier F_AB lies in the hyperplane c=0 by construction. No point on F_AB is necessarily Pareto-efficient in Ω. ∎

### The β_W Parameter

β_W = −dW/dΠ: the marginal system welfare loss per dollar of industry revenue. Calibrated via Monte Carlo simulation for each domain, using peer-reviewed externality estimates as priors.

**Iron Law:** Π is always annual *revenue* — never net profit, operating income, or margin. Profit is accounting-arbitrary; revenue is auditable and consistent across industries. Using profit inflates β_W by 5–20× for low-margin sectors.

### The C-Adjusted GDP Formula

```
C-Adj GDP = Conventional GDP − Σ(domain losses)

domain loss(i) = μ × β_W(i) × sector_GDP(i) × 0.15
```

Where:
- `μ` = shadow price scalar (1.0 = full social cost; adjustable via UI slider)
- `β_W(i)` = welfare cost coefficient for domain i (from SAPM working papers)
- `sector_GDP(i)` = country's GDP attributable to domain i (World Bank WDI, 2020–2024)
- `0.15` = net appropriation fraction: share of sector revenue representing private capture of system welfare (calibrated to SAPM median across 61 domains; see Methodology)

### Secondary: Domain-specific exposure data

For domains not directly captured by WDI sectoral data, country-level exposure (% of GDP) is estimated from:
- WHO Global Tobacco Control Report 2023 (tobacco)
- Euromonitor International 2023 (UPF, gambling)
- H2 Gambling Capital 2023 (gambling)
- ICAO/IATA Industry Statistics 2023 (aviation)
- World Bank/FAO 2023 (fisheries)
- OECD Statistics 2023 (student loans, gig economy, water privatization)

See `METHODOLOGY.md` for full source citations and derivation of each country-level exposure estimate.

## Sector Decomposition Rationale

World Bank sectoral data is mapped to SAPM domains via the following decomposition ratios. These are the most significant assumptions in the model.

| WB Variable | → SAPM Domain | Ratio | Justification |
|-------------|---------------|-------|---------------|
| `oil + gas rents %` | Oil & Gas | 1.0 | Direct rent data = sector GDP share |
| `coal rents %` | Coal | 1.0 | Direct rent data |
| `mineral rents %` | Mining / DSM | 1.0 | Direct rent data |
| `forest rents %` | Deforestation | 1.0 | Direct rent data |
| `military %` | Arms & Defense | 1.0 | Direct expenditure data |
| `agriculture % × 0.5` | Monoculture | 0.5 | ~50% of ag sector is monoculture-dependent (FAO 2022) |
| `agriculture % × 0.3` | Topsoil Erosion | 0.3 | ~30% of ag GDP linked to topsoil-mining practices |
| `manufacturing % × 0.08` | Cement | 0.08 | Cement = ~8% of manufacturing value added globally (USGS) |
| `manufacturing % × 0.05` | Chemical/POPs | 0.05 | Chemical sector = ~5% of manufacturing (UNIDO) |
| `manufacturing % × 0.02` | PFAS Exposure | 0.02 | PFAS-producing subsectors ≈ 2% of manufacturing |
| `manufacturing % × 0.01` | Nuclear | 0.01 | Nuclear generation ≈ 1% of industrial value added |
| `services % × 0.05` (max 5%) | Big Tech | 0.05 | Platform revenue ≈ 5% of services GDP (OECD digital economy) |
| `services % × 0.02` (max 2%) | Frontier AI | 0.02 | AI infrastructure spend ≈ 2% of services (Gartner 2023) |
| `services % × 0.02` (max 2%) | Algorithmic Pricing | 0.02 | Pricing technology ≈ 2% of services GDP |
| `health expenditure % × 0.05` (max 0.5%) | AMR | 0.05 | AMR-attributable health spending fraction (WHO 2019) |

All country-specific exposure data (tobacco, UPF, gambling, etc.) sourced from industry and government statistics as cited in METHODOLOGY.md.

## Replication

### Step 1: Clone and install

```bash
git clone https://github.com/epostnieks/c-adjusted-gdp.git
cd c-adjusted-gdp
npm install
```

### Step 2: (Optional) Refresh World Bank data

```bash
python fetch_gdp_data.py > gdp_data.json
```

The `gdp_data.json` in the repo is the current data snapshot (2020–2024 vintage). Running this script fetches fresh data from the World Bank API. The `RAW` constant in `src/App.jsx` contains the processed subset used by the app — run `fetch_gdp_data.py` and manually update App.jsx if you want to regenerate from scratch. A scripted pipeline connecting the two is a known limitation (see below).

### Step 3: Run locally

```bash
npm run dev
```

### Step 4: Challenge a parameter

Every β_W value is in `SAPM_BETAS` at the top of `src/App.jsx`. Every decomposition ratio is in the `compute()` function. Change any value and the entire 190-country ranking updates instantly. The shadow price μ is adjustable in the UI.

## Known Limitations

1. **No automated data pipeline.** The World Bank data is stored in `gdp_data.json` and hardcoded as `RAW` in App.jsx. Updating requires running `fetch_gdp_data.py` and manually updating the React component. A scripted bridge is planned.

2. **Sector decomposition ratios are estimated.** The ratios (e.g., `mfg × 0.08` for cement) are calibrated to global averages, not country-specific. For countries with atypical industrial structures, these will over- or under-estimate.

3. **β_W values are calibrated to global industry Π.** Applying global β_W values to country-level sector data assumes the welfare cost ratio is constant across geographies. This is an approximation; country-specific β_W calibration would require national-level externality data.

4. **Country-specific domain exposure is sparse.** For domains like tobacco, UPF, and gambling, exposure data is available for ~20–30 countries; all others receive the `default` value. Expanding this coverage is a priority.

5. **The 0.15 coefficient.** This represents the estimated share of sector revenue that constitutes net private appropriation of system welfare — the fraction that "should" flow back to the system but does not. It is calibrated to the SAPM median and applied uniformly. It is the single most contestable assumption in the model.
